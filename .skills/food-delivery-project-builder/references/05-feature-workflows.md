# 功能业务流程（全量）

本文覆盖 `04-module-features.md` 中所有功能点，按模块拆分为可执行流程。

## 1) user 模块

### 1.1 手机号注册与密码注册

1. 接收 `mobile,password,nickname`。
2. 校验手机号格式、密码强度与必填项。
3. 查询手机号是否已注册。
4. 使用安全哈希算法加密密码。
5. 写入 `user` 表，初始化 `status=1`。
6. 返回 `userId`。

### 1.2 登录并签发 JWT

1. 接收 `mobile,password`。
2. 查询用户并校验密码哈希。
3. 校验用户状态是否可登录。
4. 生成 JWT（包含 `userId`、过期时间、签名）。
5. 返回 `token,expireAt,userInfo`。

### 1.3 查询当前用户信息

1. 从 JWT 中解析 `userId`。
2. 查询用户基础信息。
3. 过滤敏感字段（如密码哈希）。
4. 返回用户资料。

### 1.4 地址增删改查

1. 新增：校验字段后写入 `user_address`。
2. 查询：按 `userId` 返回地址列表。
3. 修改：校验地址归属后更新。
4. 删除：校验地址归属后删除或逻辑删除。

### 1.5 默认地址切换

1. 接收目标地址 ID。
2. 校验地址归属当前用户。
3. 事务内清除旧默认地址。
4. 设置新默认地址 `is_default=1`。
5. 返回最新地址列表。

## 2) merchant 模块

### 2.1 商家列表分页与关键字搜索

1. 接收 `keyword,pageNo,pageSize`。
2. 组装查询条件（名称模糊匹配、营业状态过滤）。
3. 分页查询并按推荐规则排序。
4. 返回分页数据。

### 2.2 商家详情查询

1. 根据 `merchantId` 查询商家。
2. 读取评分、月售、起送价、配送费等信息。
3. 若缓存命中则直接返回，否则回源并写缓存。
4. 返回商家详情。

### 2.3 商家营业状态校验

1. 在下单/加购前读取商家状态。
2. 若 `status != 营业`，直接拒绝后续流程。
3. 返回可售或不可售原因。

## 3) product 模块

### 3.1 按商家查询分类列表

1. 接收 `merchantId`。
2. 查询 `product_category`，按 `sort` 排序。
3. 返回分类列表。

### 3.2 按商家/分类/关键字查询商品列表

1. 接收 `merchantId,categoryId,keyword,pageNo,pageSize`。
2. 过滤上架状态商品。
3. 分页查询并返回库存、销量、价格等摘要信息。

### 3.3 商品详情查询

1. 根据 `productId` 查询商品。
2. 返回商品详情（描述、价格、库存、状态）。
3. 结合缓存减少数据库读压力。

### 3.4 商品库存与上下架状态校验

1. 下单前校验商品 `status=上架`。
2. 校验库存是否满足购买数量。
3. 不满足时返回库存不足或不可售错误码。

## 4) cart 模块

### 4.1 添加商品到购物车

1. 校验用户、商家、商品状态。
2. 查询是否已有同商品购物车项。
3. 若存在则累加数量，否则新增购物车项。
4. 记录 `unit_price_snapshot`。
5. 返回购物车最新项。

### 4.2 更新数量与勾选状态

1. 根据购物车项 ID 查询并校验归属。
2. 更新 `quantity` 和 `checked`。
3. 返回更新后的购物车明细。

### 4.3 删除购物车项

1. 校验购物车项归属。
2. 删除目标购物车项。
3. 返回删除结果。

### 4.4 清空指定商家购物车

1. 接收 `merchantId`。
2. 按 `userId + merchantId` 删除购物车项。
3. 返回清空结果。

### 4.5 购物车价格汇总计算

1. 加载勾选购物车项。
2. 拉取商品最新价格与有效状态。
3. 计算小计、总额、配送费、优惠额、应付金额。
4. 返回购物车页与下单预览统一计价结果。

## 5) order 模块

### 5.1 订单预览试算

1. 接收 `merchantId,addressId,cartItemIds,couponId?`。
2. 校验地址归属与购物车项有效性。
3. 按商品最新价格重算金额。
4. 应用优惠券规则并计算应付金额。
5. 返回试算明细。

### 5.2 基于购物车项创建订单

1. 用户提交创建订单请求。
2. 校验用户、地址、商家、商品状态与库存。
3. 事务内创建 `order` 与 `order_item`。
4. 扣减商品库存（乐观锁或条件更新）。
5. 清理已下单购物车项。
6. 发送 `order.created` 事件。
7. 返回 `orderId,orderNo,payAmount,timeoutAt`。

### 5.3 订单详情查询

1. 根据 `orderId` 查询订单主表。
2. 查询订单明细项。
3. 组装状态、金额、配送信息后返回。

### 5.4 用户订单分页查询

1. 根据 `userId` 与可选 `status` 查询订单。
2. 按创建时间倒序分页。
3. 返回订单摘要列表。

### 5.5 订单取消（用户取消 + 超时取消）

1. 校验订单归属与当前状态是否允许取消。
2. 更新订单状态为已取消并记录原因。
3. 回补库存。
4. 关闭支付记录（若存在且未完成）。
5. 发送 `order.cancelled` 事件。

### 5.6 订单状态流转控制

1. 仅允许合法流转：待支付 -> 已支付 -> 已接单 -> 配送中 -> 已完成。
2. 取消态不可逆。
3. 每次流转记录操作时间与操作来源。

## 6) coupon 模块

### 6.1 秒杀券列表与倒计时

1. 查询可参与秒杀券。
2. 返回库存、开始/结束时间。
3. 前端根据时间字段渲染倒计时。

### 6.2 秒杀请求受理

1. 接收 `couponId,requestId`。
2. 调用 Redis Lua 原子校验：
   - 库存是否充足
   - 是否重复参与
   - 幂等是否已处理
3. 成功后投递 Kafka `seckill.request`。
4. 返回排队中或受理成功。

### 6.3 秒杀结果轮询

1. 用户按 `couponId` 查询秒杀结果。
2. 返回 `processing/success/fail`。
3. 失败时返回失败原因（无库存、重复、系统异常等）。

### 6.4 一人一单校验

1. Redis 侧用用户标记快速拦截重复请求。
2. DB 侧用唯一约束 `coupon_id + user_id` 兜底。
3. 任一侧判重命中即拒绝请求。

### 6.5 秒杀库存预占与补偿

1. Redis 预扣库存用于快速失败与高并发承接。
2. 消费端落库成功后确认最终状态。
3. 落库失败触发补偿：回滚库存与用户标记。
4. 对账任务定时校正 Redis 与 DB 差异。

## 7) payment 模块

### 7.1 创建模拟支付记录

1. 接收 `orderId,channel`。
2. 校验订单状态为待支付。
3. 生成 `payNo` 并写入 `payment_record(status=待支付)`。
4. 返回 `payNo,payUrl,expireAt`。

### 7.2 处理模拟支付回调

1. 接收回调参数 `payNo,status,paidAt,signature`。
2. 验签并校验回调来源。
3. 幂等检查：同一 `payNo` 只处理一次。
4. 更新支付记录状态与回调报文。
5. 若支付成功，联动更新订单为已支付并发送 `order.paid` 事件。

### 7.3 按订单查询支付状态

1. 根据 `orderId` 查询 `payment_record`。
2. 返回支付状态与关键时间字段。

### 7.4 支付回调幂等处理

1. 以 `payNo` 作为业务幂等键。
2. 对重复回调直接返回成功确认，不重复执行业务。
3. 记录重复回调日志用于审计。

## 8) common 与 infrastructure 模块

### 8.1 统一返回结构封装

1. 所有接口统一返回 `Result<T>`。
2. 包含 `code/message/data/requestId/timestamp`。
3. 业务异常映射标准错误码。

### 8.2 全局异常处理

1. 捕获参数校验异常、业务异常、系统异常。
2. 转换为统一错误响应。
3. 记录异常日志与 traceId。

### 8.3 请求日志与链路追踪 ID

1. 请求入口生成或透传 `requestId/traceId`。
2. 日志打印请求参数、响应摘要、耗时。
3. 在跨线程与 MQ 消费中传递 trace 信息。

### 8.4 Redis 缓存工具能力

1. 统一 Key 命名规范。
2. 封装读取、写入、删除、TTL、批量操作。
3. 内置缓存击穿/穿透基础策略（空值缓存、随机过期）。

### 8.5 Lua 执行封装

1. 统一加载 Lua 脚本。
2. 参数绑定与返回码标准化。
3. 用于秒杀库存与资格原子校验。

### 8.6 Kafka 生产消费事件处理

1. 生产端统一事件格式与 key 规则。
2. 消费端实现重试、死信、幂等控制。
3. 关键事件：`order.created`、`order.paid`、`order.cancelled`、`seckill.request`。

### 8.7 JWT 解析与鉴权过滤器

1. 解析请求头 JWT。
2. 验证签名、过期时间、用户身份。
3. 将用户上下文注入当前请求线程。
4. 鉴权失败统一返回未认证错误码。
