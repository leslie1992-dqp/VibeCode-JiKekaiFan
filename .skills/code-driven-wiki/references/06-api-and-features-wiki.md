# 功能与 API Wiki

## 文件位置

- 主文件：`wiki/api/features-and-apis.md`。

## 内容要求

### 1. 全量 API 清单

以表格或分模块列表形式维护，至少包含：

| 列 | 说明 |
|----|------|
| 模块 | 如 user、order |
| 功能名称 | 中文可读名称 |
| 方法 | GET/POST/PUT/DELETE |
| 路径 | 含版本前缀，如 `/api/v1/...` |
| 鉴权 | 是否需要 JWT 等 |
| 说明 | 一句话 |

### 2. 功能 ↔ API 映射

对每个业务功能（与 `wiki/features/` 下页面一一对应），列出：

- 功能 slug 或页面路径
- 涉及的主要 API（可多行）
- 关键错误码（可选）

### 3. 同步规则

- Controller 路由、方法或路径前缀变更时，更新本文件。
- 新增功能时：先补 `wiki/features/...` 流程页，再在本文件中增加一行映射。
- 若项目生成 OpenAPI，可在本文件顶部增加 **「OpenAPI 链接或生成命令」**，但 **仍以当前仓库实际代码为准** 校验表格。

## 与代码评审的配合

PR 描述中可写：「已更新 `features-and-apis.md` 第 X 节」，便于审查者核对。
