# Impeccable — 项目设计说明

本文记录 **即刻开饭** 的设计语境，便于后续会话与产品意图、视觉语言保持一致。

## Design Context（设计语境）

### Users（用户）

- **从代码推断：** 在网页端浏览与点餐的终端用户（`lang="zh-CN"`，文案为简体中文）。典型任务：找商家、对比评分/月售/距离、管理购物车、下单、使用优惠券/秒杀、查看订单。场景常带时效（饿了、要快决策）与价格敏感（界面突出配送费、优惠、合计）。
- **尚待确认：** 主要受众是演示/学习、真实区域上线还是内部工具；设备假设（移动优先还是桌面优先）。

### Brand Personality（品牌气质）

- **从代码推断：** 产品名 **即刻开饭** 传达 **快**、**日常省事**、**直白** —— 不是高端餐饮定位。界面语气偏功能、好懂（搜索、排序、运费、库存、订单状态），而非玩梗或杂志风。
- **尚待确认：** 你用三个词概括的品牌；在「快而清楚」之外的情绪目标（例如冷静可信 vs 促销感强）。

### Aesthetic Direction（审美与实现）

- **从代码推断：**
  - **技术栈：** Vue 3 + TypeScript + Vite；样式集中在 `frontend/src/styles.css` 的 **全局 CSS**（`package.json` 中无 Tailwind/组件库）。新界面应与现有 class 习惯一致，除非你刻意引入新体系。
  - **主题：** 当前样式为 **仅浅色**（页面灰底 `#f3f4f6`，白底卡片，正文色 `#1f2937`）。
  - **色彩：** 链接与主操作蓝（`#2563eb`、`#1d4ed8`）；**红** 用于价格与破坏性操作（`#dc2626`）；**绿** 用于正向交易动作（`#16a34a`）；**琥珀/橙** 强调券与秒杀（`#b45309`、`#ea580c`、黄渐变区块）。
  - **字体：** `body` 为 `Arial, sans-serif`；层级主要靠字号/字重，尚未使用自定义字体栈。
  - **布局：** 卡片列表、多处约 8px 节奏、卡片/按钮圆角约 8px；商家网格与店铺区信息偏密但可读。
- **尚待确认：** 希望参考或避免的应用/站点；必须/禁止使用的颜色；是否需要深色模式或正式的设计 token 层。

### Design Principles（设计原则）

1. **任务优先、一眼能懂** — 帮用户顺畅完成 浏览 → 购物车 → 支付 → 跟踪；价格、运费、订单状态保持可扫读。
2. **商业语义一致** — 红表示金额/危险，绿表示「进行/加购/支付成功」，蓝表示导航/主操作，琥珀色表示优惠 —— 与 `styles.css` 已有模式对齐。
3. **朴素 CSS、同一套说法** — 优先扩展现有类名与命名方式，再考虑新范式；避免每个页面各一套视觉。
4. **简体中文与本地化** — 文案、`lang`、货币（¥）、数字格式对 zh-CN 用户保持统一。
5. **可访问性基线** — 延续可聚焦控件、已有 `aria-*`、文字与按钮对比度足够；若承诺 WCAG 等级，再有意识地上探。
